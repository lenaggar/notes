Everything in linux is a file.

# Common file system commands
- `ls` list content of a directory
- `cd` change directory
- `pwd` print working directory
- `touch` create a file
- `mkdir` create a directory (`-p` creates parents of a nested path if they don't exist)
- `rm` remove a file or a directory
	- `rm -i` interactive delete with a confirmation for double checking
	- `rm -r` removes a dir and everything inside it
	- `rm -rf` this is dangerous as it **force** removes without any confirmation
- `rmdir` removes directory only if it is empty, safer than `rm -r`
- `cp` copies a file to another file/directory
	- `cp file1.md file2.md`
	- `cp file1.ms my-folder/`
- `mv` moves a file from one location to another, or can also be used to rename a file
- `find` finds a file/directory
	- `find /home -name "*.txt"` returns all files in home directory that have a .txt extension
	- `find /var -type d -name "log"` returns all directories (`-type d`) with name `log`
	- `find /var/log -size +10M` returns files inside `/var/log` that are larger in size than 10 megabytes (`-size +10M`)
- `locate` a faster command than `find` because it uses a local database to search from. The local database could be outdated and can be updated by running `sudo updatedb`.
- `which` outputs the path of a command
	- `which ls` outputs exactly where the location of the binary of `ls` program lives
- `command -v` similar to `which` but more reliable and POSIX compliant
- `type` returns what type of command
	- `type cd` => `cd is a shell builtin` because it is a command built in the shell application
	- `type ls` => `ls is /usr/bin/ls` because it is a separate application inside the user's `/usr/bin` directory
- `man` open the manual page of a command (`man ls`).
	- A few things to know how to navigate the `less` page of a manual: ^less-navigation
		- `Space` or `f` next page
		- `b` previous page
		- `/{pattern}` like `/all` to search for text inside the manual
		- `n` next search result
		- `q` quit
	- `man 5 passwd` go to page 5 for `passwd` command
	- `man -k passwd` display all relevant pages for a command or if you don't know the exact command you're looking for
- `help` similar to `man` but for shell built in commands
	- `help cd` because `cd` is a shell built in command not a user command, remember!

## Shortcuts
- `/` root directory
- `~` home directory
- `.` current directory
- `..` parent directory
- `-` previous directory

## Listing files
When doing `ls -l` it prints out the list of items inside a directory in a long format (`-l`). It's important to know that the first character indicates different things about the item:
- `-` = regular file
- `d` = directory
- `l` = symbolic link

## Displaying a tree representation of a directory
When displaying a tree structure of a directory using `tree` it goes as deep as the nesting goes. But it can be limited to how many levels deep it can go by `tree -L 2` for example for displaying a 2 levels deep nested tree but not more than 2.

## TL;DR about documentation of commands on CLI
- `man ls` => full documentation for a command
- `help cd` => shell builtin documentation
- `ls --help` => quick reminder for the command
- `man -k kayword` => don't know exactly the command name
