`tmux`  stands for terminal multiplexer

The problem it solves is basically that we can have multiple terminals for a single SSH connection, because a single process per terminal per SSH connection that is not convenient

Imagine wanting to:
- watch logs in one terminal
- edit files in another
- run commands on a third
Without `tmux` we would need 3 SSH sessions

What `tmux` solves:
- sessions persist - disconnect or reconnect without losing work
- multiple panes - split terminal into sections
- multiple windows - like browser tabs

## Commands
- `tmux` or `tmux new` => start a new tmux session
- `ctrl+b d` => detaches the current tmux session
- `tmux ls` => lists active sessions
- `tmux attach` => attach back to the most recent session (a detached one)
- `tmux attach -t 2` => attach back to the a specific session
- `tmux new -s work` => create a new session with a name
	- `tmux attach -t work`
- `tmux kill-session -t work` => kills a session

### Windows commands (similar to browser tabs)
- `ctrl+b c` => create a new window
- `ctrl+b n` => next window
- `ctrl+b p` => previous window
- `ctrl+b 0-9` => switch to window by number
- `ctrl+b ,` => rename window
- `ctrl+b &` (or type `exit`) =>  close window

### Panes commands
- `ctrl+b "` => split horizontal
- `ctrl+b %` => split vertical
- `ctrl+b arrow` => move between panes
- `ctrl+b x` (or type `exit`) => close pane
- `ctrl+b z` => toggle full-screen
- `ctrl+b ctrl+arrow` => resize panes
