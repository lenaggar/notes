**Goal**: View file contents, search for text, and understand file information.

# Viewing file contents
## cat
`cat` prints the file contents to the shell:
- `cat /etc/hostname` => prints the name of the host (machine) the user is using
- `cat /etc/os-release` => prints information about the operating system

## less
`less` prints the file content in a paginated way (pager) that you can navigate through much easily, just like we did in [the `man` command navigation](01.03_the-linux-file-system.md#^less-navigation)
- `less /var/log/syslog` => returns a less pager that the user can navigate

## head
`head` prints the first portion of a file, not the full contents like cat
- `head /var/log/syslog` => prints the first 10 lines of the file
- `head -20 /var/log/syslog` => prints the first 20 lines of the file
- `head -1 /var/log/syslog` => prints the first line of the file

## tail
`tail` prints the last portion of a file, not the full contents like cat
- `tail /var/log/syslog` => prints the last 10 lines of the file
- `tail -50 /var/log/syslog` => prints the last 20 lines of the file
- `tail -f /var/log/syslog` => prints a real-time representation of the last 10 lines of the log file as they are updated. `-f` means follow. This is essentially how logs are watched live. `ctrl+c` to exit the live watch.

## grep
`grep` searches for a pattern in a long text
- `grep "error" /var/log/syslog` => prints all the lines that have the word `error` in them
- `grep "root" /etc/passwd` => prints `root:x:0:0::/root:/usr/bin/bash`
- `grep -i "error" /var/log/syslog` => prints all the lines that have `error` in the, but also `ERROR`, or `Error` because the `-i` makes the search case-insensitive
- `grep -v "error" /var/log/syslog` => prints all the lines that **DO NOT** have `error` in them (inverse searching)
- `grep -r "password" /etc/` => does recursive search `-r` for all the files nested under `/etc/` directory
- `grep -c "error" /var/log/syslog` => prints the line count, the number of lines that match
- also regex:
	- `grep "^root" /etc/passwd` => begins with
	- `grep "bash$" /etc/passwd` => ends with
	- `grep "user[0-9]" /etc/passwd` => `user0`, `user1`, etc

## file
`file` prints what type of file it is passed
- `file /etc/passwd` => examines the content of the file to determine its type, and does **NOT** rely on file extension

## stat
`stat` prints some more information about a file, including size, permissions, owner, timestamps, and more... (`stat /etc/passwd`)

## wc
`wc` prints the count of contents of a file including lines, words, bytes.
- `wc -l /etc/paswd` => prints the number of lines in a file
- `wc -w /etc/paswd` => prints the number of words in a file
- `wc -c /etc/paswd` => prints the number of characters (bytes) in a file

## du
`du` stands for **d**isk **u**sage. the `-s` to summarize, and the `-h` to make it human-readable
- `du -sh ~/*` => prints a summary of each directory in the root home directory of the current user
- `du -h --max-depth=1 /var` => prints the disk usage of all root directories under `/var` in a human readable format

## df
`df` stands for **d**isk **f**ree. it prints how much disk is free
- `df -h` => gives a table of all mounted filesystems and their size, usage, available, and where they are mounted
