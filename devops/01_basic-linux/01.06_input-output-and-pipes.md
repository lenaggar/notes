Every program has 3 standard streams `stdin`, `stdout`, and `stderr`

| Stream   | Name            | File descriptor | Purpose                                   |
| -------- | --------------- | --------------- | ----------------------------------------- |
| `stdin`  | Standard Input  | 0               | where input comes (keyboard usually)      |
| `stdout` | Standard Output | 1               | where normal output goes (usually screen) |
| `stderr` | Standard Error  | 2               | where error messages go (usually screen)  |

# Unix filters

## |
`|` is an operator that _redirects_ the `stdout` of one program and inserts it as the `stdin` of another program

## >
`>` is an operator that _redirects_ the `stdout` of one program and inserts it to a file overwriting its contents

## >>
`>>` is an operator that _redirects_ the `stdout` of one program and inserts it to a file appending to whatever was there
- this is how log files are created

| This is where `echo` becomes useful, that we take some string and insert it or append it to a file

## <
`<` is an operator that _redirects_ the `stdout` the contents of a file inputs into a program instead of the keyboard

## 1>
`1>` is an operator that _redirects_ **only** the `stdout` of one program into a file

## 2>
`2>` is an operator that _redirects_ **only** the `stderr` of one program into a file

## &>
`&>` is an operator that _redirects_ **both** the `stdout` and `stderr` of one program into a file

## /dev/null
`/dev/null` is a black hole that means that we want to discard the output and we don't care about it
- `ls /etc > /dev/null` => discard the normal output
- `ls /etc 2> /dev/null` => discard the error output

# Some useful tricks to do with unix filters

- `history | grep "ssh"` => filters previous command
- `history | grep "docker" | ` => counts how many times "docker" command was used form command history
- `who | sort` => who is logged in sorted
- `ls /etc | wc -l` => count files in a directory
- `du -sh /var/log/* | sort -h` => list large log files by size in a human readable format
- `du -sh /var/log/* | sort -h | tail -10` => list the 10 largest log files
- `ps aux | grep "alacritty"` => search and filter for running processes
- `sort names.txt | uniq` => sorts a list of names and removes duplicates returning a unique list
- `sort names.txt | uniq -c` => sorts a list of names and removes duplicates returning a unique list with a count of each one indicating how many occurrences
- `cut -d: /etc/passwd -f1,3 | sort | uniq` => applies pattern matching to some text defining a delimiter and the fields to extract, in this case field 1 is the username and field 3 is the user id
- `echo "hello world" | cut -c1-5` => cuts text starting from character 1 till character 5 returning `hello` in this case
- `echo "heLLo" | tr 'a-z' 'A-Z'` => transform text into capital case
- `echo "heLLo" | tr 'A-Z' 'a-z'` => transform text into small case
- `echo "hello world" | tr ' ' '\n'` => transform pace characters in a text into new line
- `echo "abc123" | tr -d '0-9'` => transforms a text by deleting numbers (could also be any range of numbers `echo "abc123" | tr -d '2-9'` keeping 0s and 1s)
- `ls /etc | tee listing.txt | head -5` => `tee` writes to the screen and to a file at the same time and in this case what was going to the screen is piped into `head -5` to display only the first 5 lines
- `cat names.txt | xargs echo "Hello:"` => `xargs` builds a command from input, in this case it will take each line from `names.txt` and space join it with the initial command `echo "Hello:"`, so if the file contained two names on two lines `Mahmoud\nSara`, the output becomes `echo "Hello:" "Mahmoud" "Sara"`
- `cat names.txt | xargs -n 1 echo "Hello:"` => `xargs -n 1` builds a command from input, in this case it will take each line from `names.txt` and call the command with it, so the command is called for each line, it can be more `-n 2` one command call for each pair of lines and so on
