A process is a running instance of a program.

Every process has:
- a `pid` (process id) - unique number to that process
- a `ppid` (parent process id) - the id of the process that started it
- an `owner` - the user running it
- a `state` - running, sleeping, stopped, etc.

## Common commands
- `ps` => lists running processes attached to the terminal
- `ps aux` => list all processes running on the system
	- `a` =>  all users
	- `u` => user-oriented format
	- `x` => include processes without terminal
- `ps aux | grep "docker"` => search for processes
- `pgrep "docker"` => search for process, returns only the `pid`
- interactive process viewers:
	- `top` => old school interactive process viewer
	- `htop` => a bit newer interactive process viewer
	- `btop` => an even cooler interactive process viewer
	  in all the above, we can kill a process for example by typing `k`, then entering the `pid` to kill, then select the kill signal (more on that below)
- `kill 1234` => kills a process, takes a `pid` as param, this sends `SIGTERM` (polite request to stop, so the process can do any cleanup)
- `kill -9 1234` => kills a process, takes a `pid` as param, this sends `SIGKILL` (force kill, this should be used when the process is stuck for some reason and ignores `SIGTERM`)
- `killall firefox` => kill a process by name, in this case, kill all the processes that are related to (named) firefox
- `sleep 60 &` => put command in the background `&` after the command tells the terminal to put the process in the background
- `jobs` => command to see commands in the background and we can see its `pid` and name
- `fg %1` => bring job #1 to the foreground 
- `uname -a` => prints basic kernel/system information
- `uptime` => check how long a system has been up and running 
- `free -h` => how much memory usage we have
- `lscpu` => shows CPU architecture, cores, threads, speed
- `lscpu | grep -E "^(CPU\(s\)|Model name)"` => just the essentials of CPU info
- `lsblk` => shows block devices (disks) and partitions
- `df -h` => shows mounted filesystems and their disk usage
- `pstree` => shows a tree of processes and their relations which process is parent to which process and so on

## Systemd
- It is the first process that the kernel runs on linux and it always has `pid` of 1
- Starts services at boot
- Manages running services
- Handles dependencies between services
- Collects logs
- `systemctl` => system control, the main command to interact with systemd
- `systemctl status ssh` => checks the status of `ssh` service, and it shows some information such as:
	- title of man page documentation
	- status of the process
	- `pid`
	- how many tasks are running
	- memory usage
	- cpu usage, and
	- a small subset of logs
- `systemctl start ssh` => starts `ssh` service
- `systemctl enable ssh` => makes sure that on the next boot `ssh` service is started
- `systemctl disable ssh` => makes sure that on the next boot `ssh` service is not started
- `systemctl list-units --type=service` => list all loaded services
- `systemctl list-units --type=service --state=running` => list only running services
- `journalctl` => view system logs through systemd
- `journalctl -u ssh` => view system logs for a specific service
- `journalctl -f` => live view of system logs, `-f` for follow (similar to `tail -f`)
- `journalctl --since "1 hour ago"` => view of system logs from last hour
- `journalctl -b` => view of system logs only from current boot session
