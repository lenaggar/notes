Let's first explain some tools that are used for processing files and pipelines
```bash
#!/bin/bash

# sed - search and replace
echo "hello" | sed 's/hello/hi/'

# awk - extract fields
echo "one two three" | awk '{print $2}'

# tr - translate characters
echo "hello" | tr 'a-z' 'A-Z'

# cut - extract columns
echo "a:b:c" | cut -d: -f2
```
These are powerful, but inside bash scripts there's often better ways to achieve such operations in a better way.
> _"There's nothing that says I'm a noob more than using `sed`, `awk`, `tr`, or `cut` inside a bash script"_

So, for instance, compare these two approaches
```bash
#!/bin/bash

# External command (spawns new process)
filename=$(echo "document.txt" | sed 's/.txt//')
echo ${filename}

# Parameter expansion (built-in, faster)
filename="document.txt"
echo "${filename%.txt}"
```

Parameter expansion is better because:
- faster - no subprocess
- more portable - built into bash
- more professional - shows bash more

Every `sed` or `awk` call spawns a new process. In a loop of 1000 files, that's 1000 subprocesses spawned. Parameter expansion is instant because it is built into bash.

Default values
```bash
#!/bin/bash

# (:-) use default value if variable is empty or unset
name=""
echo "Hello, ${name:-User}"  # Hello, User
echo "${name}"               #

# (:=) set value if variable is empty or unset
name=""
echo "Hello, ${name:=User}"  # Hello, User
echo "${name}"               # User
```

Example (setting default values for missing arguments)
```bash
#!/bin/bash

name="${1:-world}"  # check first argument, if not set => use "world"
log_level="${LOG_LEVEL:-info}"  # check LOG_LEVEL, if not set => use "info"
echo "Hello, ${name} (log: ${log_level})"
```
So in this case:
`./loglevel` => `Hello, world (log: info)`
`./loglevel Mahmoud` => `Hello, Mahmoud (log: info)`
`export LOG_LEVEL=2`
`./loglevel Mahmoud` => `Hello, Mahmoud (log: 2)`

Example (setting default values for missing arguments)
```bash
#!/bin/bash

name="${1:?First parameter is required}"  # check first arg, error if unset
log_level="${LOG_LEVEL:-info}"  # check LOG_LEVEL, if not set => use "info"
echo "Hello, ${name} (log: ${log_level})"
```

Parameter expansion pattern:

| Syntax            | Meaning                                      |
| ----------------- | -------------------------------------------- |
| `${var:-default}` | use default if empty/unset, don't assign var |
| `${var:=default}` | use default if empty/unset, assign var       |
| `${var:?error}`   | error if empty/unset                         |

Example (checking password length)
```bash
#!/bin/bash

password="secretpass"
echo "Length: ${#password}"  # `#password` returns the characters count of password

# validation
if [[ ${#password} -lt 8 ]]; then
	echo "Password is too short"
	exit 1
fi
```

Remove suffix
```bash
#!/bin/bash

filename="document.txt"
echo "${filename%.txt}"  # document

filepath="backup.2024.tar.gz"
echo "${filepath%.*}"   # backup.2024.tar (shortest match)
echo "${filepath%%.*}"  # backup (longest match)
```

Remove prefix
```bash
#!/bin/bash

filepath="/home/user/document.txt"
echo "${filepath#*/}"   # home/user/document.txt (shortest match)
echo "${filepath##*/}"  # document.txt (longest match - this is basename!)
```

To remember it easily:
> _"Look at your keyboard.
> 	`#` is left of `$` => removes from left (prefix).
> 	`%` is right of `$` => removes from right (suffix)
> 	single `#` or `%` => shortest match
> 	double `##` or `%%` => longest match"_

Example (get file parts)
```bash
#!/bin/bash

filepath="/home/user/report.tar.gz"

filename="${filepath##*/}"   # report.tar.gz
directory="${filepath%/*}"   # /home/user (dirname)
extension="${filename##*.}"  # gz
basename="${filename%%.*}"   # report

echo "File name:      ${filename}"
echo "File directory: ${directory}"
echo "File extension: ${extension}"
echo "File basename:  ${basename}"
```

Search and replace
```bash
#!/bin/bash

text="hello world world"

echo "${text/world/bash}"   # hello bash world (replace first match)
echo "${text//world/bash}"  # hello bash bash (replace all matches)
echo "${text// /}"          # helloworldworld (replace with nothing - delete!)
```

Example (manipulating `PATH`)
```bash
#!/bin/bash

echo "${PATH//:/$'\n'}"  # colons to new lines (no external subprocess needed)
```
> _"If you’re reaching for `sed` to manipulate a variable, stop. There’s probably a parameter expansion for that"_

Case transformation (without using external transformation `tr`)
```bash
#!/bin/bash

name="hello world"
echo "${name^}"  # Hello world (uppercase first)
echo "${name^^}"  # HELLO WORLD (uppercase all)

name="HELLO WORLD"
echo "${name,,}"  # hello world (lowercase all)
```

## Expansion order
1. Brace expansion: `{1..5}`, `*.{md.txt}`
2. Tilde expansion: `~` becomes `$HOME`
3. Parameter expansion: `${var}`, `${var:-default}`
4. Arithmatic expression: `$((count + 1))`
5. Command substitution: `$(command)`
6. Word splitting: Unquoted results split on white spaces
7. Pathname expansion: `*.txt` expands to matching files

```bash
#!/bin/bash

n=5

echo {1..$n}       # doesn't work - prints literal {1..5}
eval echo {1..$n}  # works (but eval is dangerous!)
seq 1 $n           # better approach
```

## Summary

| Syntax            | Description            |
| ----------------- | ---------------------- |
| `${var:-default}` | use default if empty   |
| `${var:=default}` | set default if empty   |
| `${var:?error}`   | error if empty         |
| `${#var}`         | string length          |
| `${var%pattern}`  | remove shortest suffix |
| `${var%%pattern}` | remove longest suffix  |
| `${var#pattern}`  | remove shortest prefix |
| `${var##pattern}` | remove longest prefix  |
| `${var/old/new}`  | replace first match    |
| `${var//old/new}` | replace all matches    |
| `${var^^}`        | uppercase all          |
| `${var,,}`        | lowercase all          |
