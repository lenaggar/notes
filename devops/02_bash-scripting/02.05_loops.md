## For loops
Structure
```bash
#!/bin/bash

for VAR in LIST; do
	# command to run using ${VAR}
done
```

Example (inline list)
```bash
#!/bin/bash

for name in Alice Bob Charlie; do
	echo "Hello, ${name}"
done
```

Example (files glob)
```bash
#!/bin/bash

for file in *.txt; do
	[[ -f "${file}" ]] || continue
	echo "Processing: ${file}"
done
```
This ^ is very important. We should always apply this check. If no files that match `*.txt` exist, `${file}` will literally be `*.txt`. So, we need to always guard against this case!

Example (Number range `a..b`)
```bash
#!/bin/bash

for i in {1..5}; do
	echo "Count: ${i}"
done
```

Example (Number range with step `a..b..s`)
```bash
#!/bin/bash

for i in {0..10..2}; do
	echo "Even number: ${i}"
done
```

## Looping over command outputs
> _“If your script breaks on filenames with spaces, you wrote a broken script.
> Not the user’s fault for having spaces.”_

```bash
#!/bin/bash

for file in $(ls); do  # WRONG - fails with spaces in filenames
    echo "$file"
done

for file in *; do  # CORRECT - handles spaces correctly
    echo "$file"
done
```

## While loops
```bash
#!/bin/bash

count=1
while [[ ${count} -le 5 ]]; do
	echo "Count: ${count}"
	((count++))
done
```
Use `for` when you know the list. Use `while` when you don’t - reading files, waiting for conditions.

## Reading files line by line
> _“Without `-r`, backslashes get interpreted.
> Someone’s filename has a backslash? Script breaks. Always use `-r`”_

```bash
#!/bin/bash

while read -r line; do
    echo "Line: $line"
done < /etc/hosts
```
Let's breakdown ^
- `read -r line` => reads one line into variable `line`
- `-r` => prevents backslash `\` interpretation (always use it)
- `< /etc/hosts` => redirects the file input
Always `read -r`, without it, backslashes get interpreted.
If someone's data has a backslash `\`? script breaks

Process specific fields
```bash
#!/bin/bash

# /etc/passwd format => user:x:uid:gid:info:home:shell
while IFS=: read -r user _ uid gid _ home shell; do
	echo "${user} (UID ${uid}) uses ${shell}"
done < etc/passwd
```
Let's breakdown ^
- `IFS=:` => sets field separator to `:`
- `_` => a throwaway variable for fields we don't care about
`read` splits on `IFS`. `IFS` controls word splitting.

## Loop control

Break
```bash
#!/bin/bash

for i in {1..10}; do
    [[ $i -eq 5 ]] && break
    echo "$i"
done
# Outputs: 1 2 3 4
```

Continue
```bash
#!/bin/bash

for i in {1..5}; do
    [[ $i -eq 3 ]] && continue
    echo "$i"
done
# Outputs: 1 2 4 5
```

Example
```bash
#!/bin/bash

for file in .* *; do
[[ "${file}" == .* ]] && echo "Hidden file: ${file}" && continue
	[[ -f "${file}" ]] || continue  # <- guard if matches nothing
	echo "Processing: ${file}"
done
```

## Common patterns

Example (Batch rename files)
```bash
#!/bin/bash

for file in *.txt; do
	[[ -f "$file" ]] || continue
	newname="backup_${file}"
	mv "${file}" "${newname}"
done
```

Example (process files recursively)
```bash
#!/bin/bash

while IFS= read -r -d '' file; do
	echo "Found: $file"
done < <(find . -name "*.txt" -print0)
```
The `-print0` and `-d ''` handles filenames with spaces AND newlines safely.
This is a professional pattern.
