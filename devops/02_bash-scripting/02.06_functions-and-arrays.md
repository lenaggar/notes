## Functions

Example of a very basic function
```bash
#!/bin/bash

greet() {
	echo "Hello, ${1}!"
}

greet "World"  # Hello, World!
```

So, bash functions are like scoped bash scripts
```bash
#!/bin/bash

show_info() {
	echo "First arg: ${1}"
	echo "All args: ${@}"  # <-- ShellCheck will error here
	echo "Number of args: ${#}"
}

show_info one two three
```

## Functions with global variables
By default, variables are global
```bash
#!/bin/bash

my_function() {
	name="Changed"
}

name="Original"
my_function
echo "${name}"  # Changed (oops!)
```

To fix that often unintended behavior, use `local` for local variables
```bash
#!/bin/bash

my_function() {
	local name="Changed"
	echo "Inside: ${name}"  # Changed
}

name="Original"
my_function
echo "Outside: ${name}"  # Original (oops!)
```

## Function return values
Returning exit codes
```bash
#!/bin/bash

is_even() {
	local num="${1}"
	(( num % 2 == 0 ))
}

if is_even 4; then
	echo "4 is even"
fi
```

Returning data
```bash
#!/bin/bash

get_extension() {
	local filename="${1}"
	echo "${filename##*.}"
}

ext=$(get_extension "document.txt")
echo "${ext}"  # txt
```

## Arrays
```bash
#!/bin/bash

fruits=("apple" "banana" "orange")

# Single element (zero-indexed)
echo "${fruits[0]}"  # apple

# All elements
echo "${fruits[@]}"  # apple banana orange

# Length
echo "${#fruits[@]}"  # 3

# Add element
fruits+=("grape")

# Looping over arrays
for fruit in "${fruits[@]}"; do  # using "fruits[@]" with quotes to handle elements with spaces
	echo "I like ${fruit}"
done
```

Example (function that works with an array)
```bash
#!/bin/bash

print_numbered() {
    local items=("$@")
    local i=1
    for item in "${items[@]}"; do
        echo "${i}. ${item}"
        ((i++))
    done
}

tasks=("Buy milk" "Walk dog" "Write code")
print_numbered "${tasks[@]}"
# 1. Buy milk
# 2. Walk dog
# 3. Write code
```
