- `k get pods` => lets some information about running pods
- `k get pods -o wide` => gets a lot more information about running pods
- `k delete pods nginx` => delete a pod
- `k exec -it nginx -- /bin/bash` => get an interactive shell into the default container in the pod
  if pod is running multiple containers (use `-c <container-name>`)
  as in `k exec -it nginx -c <container-name> -- /bin/bash`

We can get the ip address for pod-A in the cluster and get an interactive shell form pod-B and ping/curl pod-A

The skill for getting into a pod and inspecting it from the inside is an essential debugging skill for running kubernetes in production
